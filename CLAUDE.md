# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- Setup: `uv sync` (Python 3.12, uv-managed; run the CLI with `uv run vidcp ...`)
- Test (default for iteration): `uv run pytest -m "not slow"` — skips model-heavy tests
- Full test suite: `uv run pytest` (downloads Whisper/embedding models on first run; tests force the `tiny` Whisper model)
- Single test: `uv run pytest tests/test_search.py::test_name`
- Lint: `uv run ruff check .` — Format: `uv run ruff format .` (CI runs both with `--check`)

## Requirements & environment

- `ffmpeg` and `ffprobe` must be on PATH (`vidcp doctor` verifies).
- User data lives outside the repo in `~/.vidcp` (SQLite `library.db` + `store/`), overridable via `VIDCP_HOME`. Settings precedence: `VIDCP_*` env vars > `<home>/config.toml` > defaults.
- `get_settings()` is `lru_cache`d — tests that touch `VIDCP_*` env vars must call `get_settings.cache_clear()` (the autouse `_isolated_home` fixture handles this).
- Test fixtures `color.mp4`/`cuts.mp4`/`text.mp4` are gitignored and regenerated by `tests/fixtures/generate.py` (needs ffmpeg + Pillow); only `speech.mp4` is committed.

## Conventions

- Line length 100 (ruff); `from __future__ import annotations` at the top of every module.
- Lazy-import heavy libraries (faster-whisper, rapidocr, sentence-transformers, scenedetect, imagehash, even `vidcp.search`) inside functions — never at module top level. This keeps CLI startup fast; preserve it in new code.
- User-facing errors: raise `VidcpError(message, hint=...)` (a ClickException) so they render cleanly; tracebacks only appear under `--debug`.
- Every CLI command that prints data also offers `--json` (via Pydantic `model_dump(mode="json")`); human output uses Rich tables.
- SQL migrations are append-only plain-string DDL in `db.py` — never edit an existing migration; the list index is the schema version.
- Model-heavy tests get the `slow` pytest marker.

## Architecture notes

- Pipeline DAG (`src/vidcp/pipeline/`): `probe → {audio → transcribe}` and `{scenes → keyframes → ocr}` → `embed`. Stages are resumable with config-fingerprint invalidation (`config_hash`); changing a stage's config keys means its dependents re-run too (`transitive_dependents`).
- Video IDs are SHA-256, displayed truncated to 8 chars; any unique prefix resolves via `library.resolve_id`.
